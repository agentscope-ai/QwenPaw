## 🎯 背景和需求

### 问题描述

在使用 CoPaw Agent 执行长时间任务时（如下载大量数据、生成复杂报告、浏览多个网页等），用户无法中断当前执行的任务。这导致：

1. **用户体验差**: 即使发现指令有误，也只能等待任务完成
2. **资源浪费**: 错误任务继续执行，浪费 API 配额和时间
3. **无法及时调整**: 用户有新需求时，必须等待当前任务完成

### 实际场景

```
用户：帮我下载最近 1 年的股票数据
Agent: 开始下载...（预计需要 5 分钟）
[执行 1 分钟后]
用户：（发现只需要最近 1 个月的数据，但无法打断）
[只能等待 5 分钟完成]
用户：停下来，我只要最近 1 个月的
Agent: 好的，那我现在重新下载...（又浪费 5 分钟）
```

---

## ✨ 解决方案

添加会话打断机制，支持用户通过简单指令中断 Agent 执行：

### 支持的打断指令

- `stop`
- `停下来` (推荐)
- `停下`
- `停止`
- `别做了`
- `别继续`
- `取消`
- `中断`

### 使用效果

```
用户：帮我下载最近 1 年的股票数据
Agent: 🔄 正在连接 Tushare...
Agent: 📊 已下载 1000 只股票...
[执行中...]
用户：停下来
Agent: 🫡 已停下！

超哥，有什么需要调整的？
用户：只要最近 1 个月的
Agent: 好的，马上下载最近 1 个月的数据...
```
