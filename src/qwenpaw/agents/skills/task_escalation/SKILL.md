# 压缩版升级求助策略

当你是能力较弱的 Agent，但可以通过 `list_agents` 和 `chat_with_agent` 向更强 Agent 求助时，使用本 Skill。

调用方式：
- 使用 `list_agents()` 查可用 Agent
- 使用 `chat_with_agent(...)` 向更强 Agent 求助
- 在 `text` 中提供结构化上下文，说明你需要什么帮助
- 需要续聊时传 `session_id`

推荐调用骨架：

```text
list_agents()

chat_with_agent(
  to_agent="<stronger_agent>",
  text="[Agent <auto> requesting] ...结构化求助内容...",
)
```

## 总规则

不要所有事都求助。
也不要所有事都自己做。

只做三选一：
1. 直接回答
2. 先短暂尝试，再决定
3. 立即升级求助

## 直接回答：满足大多数即可

如果大多数条件成立，直接自己做：
- 风险低
- 任务常规或范围窄
- 输出形式明确
- 步骤依赖少
- 不需要复杂权衡
- 你对答案有把握

典型场景：
- 简单解释
- 改写、总结、翻译
- 格式整理
- 直接问答
- 小型、规则明确的转换任务

## 先短暂尝试，再决定

如果任务中等复杂，但仍有结构，可以先做一次最小尝试：
- 说明用户要什么
- 说明主要难点
- 给出你当前最可能的方向
- 说清还不确定什么

如果尝试后，关键不确定性仍然存在，就升级。

## 立即升级：满足任一即可

以下任一成立，立刻用 `chat_with_agent(...)`：
- 高风险任务
- 需要深度多步推理
- 涉及系统设计、架构、策略、复杂权衡
- 任务高度模糊，需要强判断力
- 明显超出你的可靠能力范围
- 你找不到安全可信的推进路径
- 用户明确要求深入、全面、高置信分析
- 你已经尝试过，但仍不信任自己的答案

## 硬触发条件

以下情况直接升级：
- 安全敏感建议
- 破坏性或不可逆操作
- 影响生产环境的决策
- 高风险法律/医疗/金融判断
- 多个严肃方案的比较与取舍
- 大量上下文整合
- 明显能力缺口

## 不要升级的情况

以下情况不要升级：
- 常规任务，你已经能做
- 看起来很长，但本质简单
- 真正问题只是需要向用户澄清
- 真正问题只是缺一个小事实
- 你只是为了省事想升级

## 快速打分法

每项打 0-2 分：
- 复杂度
- 风险
- 不确定性
- 能力缺口
- 判断要求

决策：
- 0-3：直接回答
- 4-5：先短暂尝试，再决定
- 6 及以上：升级

如果命中硬触发条件，直接升级，不看总分。

## 先判断你要哪种帮助

调用 `chat_with_agent(...)` 前，先选一种：
- 方向判断型：哪条路线更好？
- 审查校验型：帮我找漏洞、风险、遗漏
- 补全增强型：帮我把答案补强、补完整

## 工具调用规则

升级时默认按这个顺序执行：
1. 调用 `list_agents()` 确认可用的更强 Agent
2. 选择最合适的目标 Agent
3. 调用 `chat_with_agent(...)` 发送结构化求助内容

`chat_with_agent` 常用参数：
- `to_agent`: 目标 Agent ID
- `text`: 求助内容，建议使用结构化模板
- `session_id`: 可选，续接已有对话时传入

说明：
- `base_url` 一般不需要传，工具会自动解析当前 API 地址
- 不传 `session_id` 会自动创建新会话

## 求助模板

任务摘要：
[简要概括用户请求]

用户目标：
[最终想达成什么]

约束条件：
- [...]
- [...]

我当前的分析：
[我目前的理解和倾向方向]

我不确定的地方：
1. [...]
2. [...]

所需帮助类型：
[方向判断型 / 审查校验型 / 补全增强型]

请重点帮助我：
- [...]
- [...]

期望输出格式：
[可选]

示例：

```text
chat_with_agent(
  to_agent="strong_reasoner",
  text="""
任务摘要：
用户要我评估两个方案的长期维护成本。

用户目标：
给出一个高置信建议，并说明为什么。

约束条件：
- 不能中断现网
- 团队只有 2 名后端

我当前的分析：
方案 A 上线快，但后续扩展性一般；方案 B 改动大，但更稳。

我不确定的地方：
1. 当前场景下是否值得为长期扩展性付出更高初始成本
2. 有没有我遗漏的关键风险

所需帮助类型：
方向判断型

请重点帮助我：
- 比较两个方案的主要权衡
- 指出我可能遗漏的风险
""",
)
```

如果需要续聊：

```text
chat_with_agent(
  to_agent="strong_reasoner",
  text="请基于刚才的分析，再补充迁移风险。",
  session_id="<previous_session_id>",
)
```

## 收到帮助后

把更强 Agent 的回复当作“决策支持”，不是“直接照抄”。
只保留相关内容，再用你自己的口吻完成最终回答。

## 行为护栏

- 不要为了省事而升级
- 不要在没理解任务前就升级
- 不要把凌乱原文直接发过去，能总结就先总结
- 不要提模糊问题，能具体就具体
- 不要无脑复制对方回答

## 最终口诀

常规任务自己做。
中等任务先短试。
高风险、高不确定、超能力边界的任务再升级。
升级时发送结构化上下文。
