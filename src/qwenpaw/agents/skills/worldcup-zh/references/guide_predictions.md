# 预测系统指南

**文件：** `worldcup2026/predictions.json`（工作区根目录下的 `worldcup2026/` 目录中）
**格式：** 首次创建时使用 `references/tpl_predictions.json` 作为模板。
**规则：** 始终先 `read_file`，再 `write_file` 更新。

### 激活追踪
当用户进行**首次**预测时：
1.  正常更新 `worldcup2026/predictions.json`。
2.  读取 `worldcup2026/user_favorites.json`。如不存在，从 `references/tpl_user_favorites.json` 创建。设置 `"features_activated.predictions": true`。
3.  这确保后续回复中不再显示预测页脚。

## 计分

| 结果 | 分数 |
|---|---|
| 比分完全正确 | **+10** |
| 胜负正确，比分错误 | **+5** |
| 平局且比分正确 | **+10** |
| **平局结果正确（比分错误）** | **+5** |
| 预测错误 | **0** |

> **注意：** 不要给部分分数或"安慰"分（如 +3），除非上面明确说明。如果用户胜负判断正确（即使比分不对），为 **+5**。如果平局判断正确（比分不对），为 **+5**。

## 统计逻辑与定义

**跨会话和用户的一致性原则：**

* **待定预测规则：** 任何 `"status": "pending"` 的预测，其 `correct_exact: 0`、`correct_winner: 0`、`wrong_winner: 0`、`wrong_score: 0`、`points: null`。**不要将待定预测计入准确率百分比或错误比分的统计中。** 仅在 `actual_score` 设置且比赛结束后更新统计。

* **`correct_winner`（包含式）：** 统计用户胜负判断正确的**所有**预测（含平局结果正确）。
    *   **公式：** `correct_winner = (完全命中) + (胜负正确，比分错误)`。
    *   *示例：* 如果有 3 次完全命中，4 次接近命中，`correct_winner` 为 **7**。
* **`wrong_winner`：** 统计结果完全错误的预测。
* **`accuracy_winner_pct`：** 计算公式 `correct_winner / 已有结果的预测总数`（分母排除待定）。

* **`wrong_score`（全面统计）：** 统计所有非完全命中的已有结果预测。
    *   **包含：** "胜负错误"和"胜负正确但比分错误"两类。
    *   **公式：** `wrong_score = 已有结果的预测总数 - correct_exact`。

* **`accuracy_exact_pct`：** 用户**完全命中比分**的预测百分比。
    *   **公式：** `(correct_exact / 已有结果的预测总数) * 100`（分母排除待定）。

## 用户格式

*   "墨西哥 2-0" 或 "2-1" → 精确比分（始终包含队名或比分）
*   "2-2" 或 "1-1" → 平局比分
*   **所有预测必须包含比分。** 仅写队名（"墨西哥"）或"平局"无效。如果用户未提供比分，提示其补充。

## 预测 CTA 框

**始终**在赛前分析末尾（"关键看点"之后）包含此框：

### 标准版（首次预测）

```markdown
---

## 🎯 做出你的预测！

| 你的战绩 | 🏆 [W]胜–[L]负 · [N]次预测 · [P]分 |
|---|---|

**回复比分做出预测：**
- **`[球队A] 2-0`** 或 **`[球队B] 1-3`** — 包含你预测会赢的球队
- **`2-2`** 或 **`1-1`** — 你认为是平局

> ⚠️ **需要包含比分。** 仅写队名或"平局"不计入 — 始终包含比分。
> *终场哨响后我会立刻给你的预测打分！*
```

### 已预测版（用户已有预测）

当用户已预测此场比赛时，显示其当前选择及 ✅ 标记：

```markdown
---

## 🎯 做出你的预测！

| 你的战绩 | 🏆 [W]胜–[L]负 · [N]次预测 · [P]分 |
|---|---|

✅ **你当前的预测：** **[预测比分]**

**想修改吗？回复新的比分：**
- **`[球队A] 2-0`** 或 **`[球队B] 1-3`** — 新的预测比分
- **`2-2`** 或 **`1-1`** — 新的平局预测

> ⚠️ **需要包含比分。** 始终包含比分。
> *终场哨响后我会立刻给你的预测打分！*
```

**检测方式：** 检查 `worldcup2026/predictions.json`（工作区根目录下的 `worldcup2026/` 目录中），查找匹配的 `match` 或 `match_id` 条目。如 `predicted_score` 已设置（非 null），显示"已预测"版本。

### 验证

记录任何预测前，通过网络搜索验证比赛是否存在。
- **淘汰赛占位符：** 对于已确认的淘汰赛对阵（如"B组第一 vs D组第二"），使用占位符格式记录，待球队确定后更新。
- **重复比赛处理：** 如两支球队在赛事中多次交手（如小组赛 + 决赛重遇）：
  1.  **始终使用 `match_id`** 作为记录和计分的主键。
  2.  如用户提供"球队A vs 球队B"但未指定比赛 ID 或日期，默认取两队之间的**下一场即将到来的比赛**。
  3.  如多场比赛已结束或同时进行，请用户澄清（如"小组赛还是淘汰赛？"）。

**强制比分：** 当用户只回复队名（如"墨西哥"）或仅写"平局"时，拒绝并请其包含比分。解析比分格式：
- `球队名 X-Y` → 解析球队和比分
- `X-Y` → 仅解析比分（根据比分推断胜者，或平局如比分相等）
