# 赛后分析模板

当用户询问**已结束的比赛**时使用。

⚠️ 比赛结果/数据来自网络搜索。不要编造比分。

## ⚠️ 强制页脚 — 发送前追加

1. 读取 `worldcup2026/user_favorites.json`。如不存在，从 `references/tpl_user_favorites.json` 创建。
2. 对于以下每个为 `false`（或缺失）的标志，在 `---` 分隔符后追加对应行。每行独立，项目间空一行，不使用项目符号：

| 标志为 `false` | 追加此行 |
|-----------------|------------------|
| `features_activated.predictions` | `🎯 **预测：** 回复比分（如"墨西哥 2-1"）开启你的预测连击！` |
| `features_activated.digest` | `📰 **每日摘要：** 回复你喜欢的球队和球员，每天早上获取个性化摘要。` |
| `features_activated.video` | `🎬 **AI 视频：** 回复"做一个 [球员/球队] 的视频"生成精彩短片。` |

3. 功能成功使用后，立即更新 `worldcup2026/user_favorites.json` 将该标志设为 `true`。
4. 文件缺失或键缺失 = 显示该功能的页脚行。

## 步骤

1.  **比分预测：** 读取 `worldcup2026/predictions.json`（工作区根目录下的 `worldcup2026/` 目录中）。如用户预测了此场比赛，在回复前计算分数并更新文件。
2.  **输出：** 呈现比赛摘要。

## 回复前检查清单

生成输出前验证以下项目：

- [ ] 读取 `worldcup2026/predictions.json` 并先为用户预测打分
- [ ] 比赛数据表格中包含 `xG`
- [ ] 包含 ⭐ **最佳球员** 章节及理由
- [ ] 如存在转折点，包含 🔄 **转折点** 作为独立章节
- [ ] 包含 🏆 **意义何在** 作为独立章节（小组积分/淘汰赛影响）
- [ ] 标题必须包含：组别、场次、星期、日期、球场、城市
- [ ] 使用 ✅ emoji 作为终场标题（不要用 🏁 或其他）
- [ ] 不要超出模板添加额外章节（除非明确要求，不加红黄牌、助攻、叙事报道）
- [ ] 严格遵循以下模板结构

## 输出格式

```markdown
# ✅ 终场 — [球队A] [X] - [X] [球队B]

> [组别] • 第[N]场 • [星期]，[日期] | 📍 [球场]，[城市]

---

## ⚽ 进球

| 分钟 | 球员 | 球队 |
|---|---|---|
| [X]' | [进球者] | [球队] |

## 📊 比赛数据

| 数据 | [球队A] | [球队B] |
|---|---|---|
| 控球率 | [X]% | [Y]% |
| xG | [X] | [Y] |

## ⭐ 最佳球员

> **[球员姓名]**（[球队]）
> [理由]

---

## 🔄 转折点

[比赛转折时刻]

## 🏆 意义何在

[小组积分/淘汰赛影响]
```

## 预测得分（如适用）

如有预测，显示结果：

```markdown
🎯 **你的预测得分！**
| 比赛 | 你的预测 | 实际比分 | 结果 | 分数 |
|---|---|---|---|---|
| [A] vs [B] | [预测] | [实际] | [结果] | **[分]** |
```

## 赛后预测 CTA

比赛摘要（及任何预测得分）之后，包含一个引导用户继续参与预测的结尾提示。根据上下文选择以下变体：

### 变体 A：预测已计分（用户有预测记录）

读取 `worldcup2026/predictions.json` 获取用户当前记录。显示：

```markdown
---

🎯 你目前的战绩：**🏆 [W]胜–[L]负 · [N]次预测 · [P]分**
想预测下一场比赛吗？直接回复比分 — 终场哨响后我为你打分。
```

### 变体 B：本场未预测

```markdown
---

🎯 这场你没预测 — 没关系！下一场想锁定比分吗？回复你的选择，开球后我为你打分。
```

### 变体 C：全新用户（0 次预测，首次接触系统）

使用稍详细的语气：

```markdown
---

🎯 **我正在追踪世界杯比分预测，一起来玩吧！**
直接回复任意即将到来比赛的比分（如 `巴西 2-1` 或 `2-2` 表示平局）— 终场哨响后我立刻为你打分。
```

### 规则
- 选择与用户当前状态匹配的变体。检查 `worldcup2026/predictions.json` 中的总预测数。
- 在赛后摘要后追加。
- 保持简洁。最多一两行。
