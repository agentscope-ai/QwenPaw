# 世界杯比赛伴侣

模块化世界杯技能，路由逻辑在此，模板和指南在 `references/` 中。

## 📦 文件架构

**工作区根目录文件**（用户特定，自行维护）：
- `worldcup2026/user_favorites.json` — 收藏的球队/球员，功能开关
- `worldcup2026/predictions.json` — 用户的比分预测

**技能 `references/` 模板**（随技能分发，只读）：
- `references/tpl_predictions.json`、`references/tpl_user_favorites.json` — 首次创建文件的格式模板。
- `references/tmpl_*.md` — 赛前、赛后、直播中、每日摘要、视频、通用等输出模板。
- `references/guide_predictions.md` — 预测系统规则。

## 🧭 查询分类

* **纯球队查询：** 如果查询提到特定球队但未提及具体比赛（如 "美国队阵容"、"巴西伤病"、"韩国队表现如何？"），属于**通用话题**。搜索网络获取球队赛程和新闻，然后使用 `tmpl_general.md`。
* **比赛相关查询：** 如果查询提及特定比赛、揭幕战、决赛或赛程（如 "第一场比赛"、"第二场"、"揭幕战"、"决赛"、"第3场"、"巴西 vs 摩洛哥"、"美国队的首场比赛"）——属于**比赛相关查询**。网络搜索比赛详情，然后使用比赛模板。默认使用赛前/赛后/直播模板。
* **通用信息：** 纯信息请求（如 "有多少个小组？"、"哪些城市举办？"、"2018年谁赢了？"、"玻利维亚进世界杯了吗？"），网络搜索答案，然后使用 `tmpl_general.md`。

1.  **分类查询：**
    *   **视频请求（"做一个关于……的视频"、"生成视频"）？** → 阅读 `references/tmpl_video.md`，然后执行。
    *   **每日摘要 / "帮我补课" / "今日新闻"？** → 阅读 `references/tmpl_digest.md`，然后执行。
    *   **具体即将到来的比赛？** → 阅读 `references/tmpl_pre_match.md`，然后执行。
    *   **正在直播的比赛？** → 阅读 `references/tmpl_live.md`，然后执行。
    *   **已结束的比赛？** → 阅读 `references/tmpl_post_match.md` + 检查 `worldcup2026/predictions.json`，然后执行。
    *   **通用话题（伤病、小组等）？** → 网络搜索信息。然后阅读 `references/tmpl_general.md`，然后执行。
    *   **没有比赛上下文的预测？** → 询问是哪场比赛。

2.  **检查清单：**
    *   **比赛相关：** 验证时区（用户 + 当地），从 2 个以上来源交叉验证伤病信息，包含预测 CTA（仅赛前）。
    *   **通用查询：** 直接回答。**不**强制使用模板。注明来源。
    *   **⚠️ 数据来源：** 所有数据 — 赛程、场馆、伤病、状态、赔率、天气、突发新闻 — 必须来自网络搜索，禁止使用训练数据或记忆。

## ⚠️ 页脚

页脚（预测、摘要、视频 CTA）在各输出模板中定义。参见 `references/tmpl_general.md`、`references/tmpl_pre_match.md` 等中的 "MANDATORY FOOTER" 部分。SKILL.md 不再重复 — 模板是唯一来源。

## 通用规则

*   **文件写入：** 预测系统操作时，始终先 `read_file` 再 `write_file`，避免竞争条件。
*   **样式：** 在球队名旁使用国旗 emoji（如 🇲🇽 墨西哥、🇿🇦 南非）。
*   **时区：** 比赛时间始终同时显示**用户时区**和**当地球场时区**。
*   **海拔：** 对墨西哥城（及其他高海拔地区），提及海拔对比赛的影响。

### ⚠️ Agent 能力说明

部分功能依赖的工具可能并非所有 Agent 都可用：

| 功能 | 所需能力 | 不可用时的降级方案 |
|------|---------|-------------------|
| **每日摘要定时** | `cron` 技能或同等功能 | 用户手动触发摘要 |
| **视频生成** | Python 3 + `execute_shell_command` + `write_file` + 视频 API 密钥 | 告知用户视频生成不可用 |
| **视频显示** | `view_video` 工具 | 直接提供视频文件路径或 URL |
| **网络搜索** | `tavily_search` 或同等工具 | 使用 Agent 已有的任何搜索工具 |
| **文件操作** | `read_file` / `write_file` | 通过 shell 命令读写 |

核心技能（比赛分析、预测、通用查询）无需任何特殊工具，只需文件读写和网络搜索即可工作。
