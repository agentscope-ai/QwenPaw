Votre assistant IA personnel ; facile à installer, déployez sur votre propre machine ou dans le cloud ; supporte plusieurs applications de chat avec des capacités facilement extensibles.

> **Capacités principales :**
>
> **Chaque canal** — DingTalk, Feishu, QQ, Discord, iMessage, et plus encore. Un seul assistant, connectez selon vos besoins.
>
> **Sous votre contrôle** — Mémoire et personnalisation sous votre contrôle. Déployez localement ou dans le cloud ; rappels planifiés vers n'importe quel canal.
>
> **Skills** — Cron intégré ; Skills personnalisées dans votre espace de travail, chargées automatiquement. Pas de dépendance.
>
> <details>
> <summary><b>Ce que vous pouvez faire</b></summary>
>
> <br>
>
> - **Social** : digest quotidien des posts populaires (Xiaohongshu, Zhihu, Reddit), résumés Bilibili/YouTube.
> - **Productivité** : digest de newsletters vers DingTalk/Feishu/QQ, contacts depuis email/calendrier.
> - **Créativité** : décrivez votre objectif, lancez pendant la nuit, obtenez un brouillon le lendemain.
> - **Recherche** : suivre les actualités tech/IA, base de connaissances personnelle.
> - **Bureau** : organiser les fichiers, lire/résumer des documents, demander des fichiers en chat.
> - **Explorer** : combiner Skills et cron pour créer votre propre application agentique.
>
> </details>

## Démarrage rapide

### pip install (recommandé)

Si vous préférez gérer Python vous-même :

```bash
pip install copaw
copaw init --defaults
copaw app
```

Puis ouvrez **http://127.0.0.1:8088/** dans votre navigateur pour la Console (discuter avec CoPaw, configurer l'agent). Pour discuter dans DingTalk, Feishu, QQ, etc., ajoutez un canal dans la [documentation](https://copaw.agentscope.io/docs/channels).

![Console](https://img.alicdn.com/imgextra/i4/O1CN01iuGyNc1mNwsUU5NQI_!!6000000004943-2-tps-3822-2070.png)

### Installation en une ligne (bêta, amélioration continue)

Python n'est pas requis — l'installateur gère tout :

**macOS / Linux :**

```bash
curl -fsSL https://copaw.agentscope.io/install.sh | bash
```

Pour installer avec le support Ollama :

```bash
curl -fsSL https://copaw.agentscope.io/install.sh | bash -s -- --extras ollama
```

Pour installer avec plusieurs extras (ex. Ollama + llama.cpp) :

```bash
curl -fsSL https://copaw.agentscope.io/install.sh | bash -s -- --extras ollama,llamacpp
```

**Windows (PowerShell) :**

```powershell
irm https://copaw.agentscope.io/install.ps1 | iex
```

Puis ouvrez un nouveau terminal et exécutez :

```bash
copaw init --defaults   # ou : copaw init (interactif)
copaw app
```

**macOS / Linux :**

```bash
# Installer une version spécifique
```

### Utiliser Docker

Les images sont sur **Docker Hub** (`agentscope/copaw`). Tags d'image : `latest` (stable) ; `pre` (pré-version PyPI).

```bash
docker pull agentscope/copaw:latest
docker run -p 8088:8088 -v copaw-data:/app/working agentscope/copaw:latest
```

Également disponible sur Alibaba Cloud Container Registry (ACR) pour les utilisateurs en Chine : `agentscope-registry.ap-southeast-1.cr.aliyuncs.com/agentscope/copaw` (mêmes tags).

Puis ouvrez **http://127.0.0.1:8088/** pour la Console. La config, la mémoire et les Skills sont stockées dans le volume `copaw-data`. Pour passer des clés API (ex. `DASHSCOPE_API_KEY`), ajoutez `-e VAR=valeur` ou `--env-file .env` à `docker run`.

L'image est construite de zéro. Pour construire l'image vous-même, référez-vous à la section [Construire l'image Docker](scripts/README.md#build-docker-image) dans `scripts/README.md`, puis publiez dans votre registre.

### Utiliser ModelScope

**Pas d'installation locale ?** [ModelScope Studio](https://modelscope.cn/studios/fork?target=AgentScope/CoPaw) configuration cloud en un clic. Définissez votre Studio comme **non public** pour que d'autres ne puissent pas contrôler votre CoPaw.

### Déployer sur Alibaba Cloud ECS

Pour exécuter CoPaw sur Alibaba Cloud (ECS), utilisez le déploiement en un clic : ouvrez le [lien de déploiement CoPaw sur Alibaba Cloud (ECS)](https://computenest.console.aliyun.com/service/instance/create/cn-hangzhou?type=user&ServiceId=service-1ed84201799f40879884) et suivez les instructions. Pour des instructions étape par étape, voir [Alibaba Cloud Developer : Déployez votre assistant IA en 3 minutes](https://developer.aliyun.com/article/1713682).

## Clé API

Si vous utilisez un **LLM cloud** (ex. DashScope, ModelScope), vous devez définir une clé API avant de discuter. CoPaw ne fonctionnera pas tant qu'une clé valide n'est pas configurée.

**Où la définir :**

1. **`copaw init`** — Quand vous exécutez `copaw init`, la commande comprend une étape pour configurer le fournisseur LLM et la clé API. Suivez les invites pour choisir un fournisseur et saisir votre clé.
2. **Console** — Après `copaw app`, ouvrez **http://127.0.0.1:8088/** → **Paramètres** → **Modèles**. Sélectionnez un fournisseur, renseignez le champ **Clé API**, puis activez ce fournisseur et ce modèle.
3. **Variable d'environnement** — Pour DashScope vous pouvez définir `DASHSCOPE_API_KEY` dans votre shell ou dans un fichier `.env` dans le répertoire de travail.

Les outils nécessitant des clés supplémentaires (ex. `TAVILY_API_KEY` pour la recherche web) peuvent être définis dans la Console **Paramètres → Variables d'environnement**, ou voir [Config](https://copaw.agentscope.io/docs/config) pour les détails.

> **Vous utilisez uniquement des modèles locaux ?** Si vous utilisez les [Modèles locaux](#modèles-locaux) (llama.cpp ou MLX), vous n'avez **pas** besoin de clé API.

## Modèles locaux

CoPaw peut exécuter des LLMs entièrement sur votre machine — aucune clé API ou service cloud requis.

| Backend       | Idéal pour                                     | Installation                                                          |
| ------------- | ---------------------------------------------- | --------------------------------------------------------------------- |
| **llama.cpp** | Multiplateforme (macOS / Linux / Windows)      | `pip install 'copaw[llamacpp]'` ou `bash install.sh --extras llamacpp` |
| **MLX**       | Macs Apple Silicon (M1/M2/M3/M4)              | `pip install 'copaw[mlx]'` ou `bash install.sh --extras mlx`          |
| **Ollama**    | Multiplateforme (nécessite le service Ollama)  | `pip install 'copaw[ollama]'` ou `bash install.sh --extras ollama`    |

Après installation, téléchargez un modèle et commencez à discuter :

```bash
copaw models download Qwen/Qwen3-4B-GGUF
copaw models # sélectionnez le modèle téléchargé
copaw app # démarrer le serveur
```

Vous pouvez également télécharger et gérer les modèles locaux depuis l'interface Console.

## Installer depuis les sources

```bash
git clone https://github.com/agentscope-ai/CoPaw.git
cd CoPaw
pip install -e .
```

- **Dev** (tests, formatage) : `pip install -e ".[dev]"`
- **Console** (compiler le frontend) : `cd console && npm ci && npm run build`, puis `copaw app` depuis la racine du projet.

## Pourquoi CoPaw ?

CoPaw représente à la fois un **Co Personal Agent Workstation** (Station de travail d'agent personnel coopératif) et une « co-patte » — un partenaire toujours à vos côtés. Plus qu'un simple outil froid, CoPaw est une chaleureuse « petite patte » toujours prête à donner un coup de main (ou de patte !). C'est l'ultime coéquipier pour votre vie numérique.
