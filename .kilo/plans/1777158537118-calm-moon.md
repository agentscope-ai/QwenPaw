# План: Мульти-инстансы каналов + назначение агентов

## Цель

1. **Duplicate** — кнопка на карточке канала, создаёт копию как кастомный инстанс
2. **Hot-reload** — без перезапуска процесса
3. **Назначение агента** — дропдаун "Agent" в drawer канала, выбирает какой агент обрабатывает сообщения

---

## Как работает назначение агента

**Назначение агента на канал — опционально.** Если не выбрано — всё работает как сейчас (текущий агент обрабатывает свои каналы).

Дропдаун "Agent" в настройках канала, `allowClear` — можно очистить. При сохранении `target_agent` — process wrapper диспатчит сообщения к выбранному агенту (у него уже своя модель, промпт, инструменты). Если `target_agent` пуст/очищен — текущий агент (по умолчанию, без изменений поведения).

Пример потока:
1. Duplicate Telegram → создаётся `telegram_copy_1` (proxy в `custom_channels/`)
2. Открываю drawer `telegram_copy_1` → ввожу bot_token, allow_from
3. В дропдауне "Agent" выбираю "sales_bot" → сохраняю
4. `sales_bot` теперь обрабатывает сообщения из `telegram_copy_1` (со своей моделью и промптом)

---

## Архитектура: Duplicate через Proxy-каналы

При дублировании Telegram создаётся `~/.copaw/custom_channels/telegram_copy_1.py`:
```python
from qwenpaw.app.channels.telegram.channel import TelegramChannel

class ProxyChannel(TelegramChannel):
    channel = "telegram_copy_1"
```

- Registry автоматически обнаруживает при `from_config()` (пересканирует `custom_channels/`)
- Конфиг хранится в `__pydantic_extra__` `ChannelConfig`
- **Не нужно менять `config.py`, `manager.py`, `base.py`**
- Hot-reload: `schedule_agent_reload()` → пересоздаёт ChannelManager → пересканирует registry

```python
    "weixin":     ("weixin.channel", "WeixinChannel"),
    "onebot":     ("onebot.channel", "OneBotChannel"),
}

TEMPLATE = '''
from qwenpaw.app.channels.{module} import {cls}

class ProxyChannel({cls}):
    channel = "{key}"
'''

def generate_proxy(source_key: str, new_key: str) -> Path:
    module, cls = CHANNEL_CLASS_MAP[source_key]
    CUSTOM_CHANNELS_DIR.mkdir(parents=True, exist_ok=True)
    path = CUSTOM_CHANNELS_DIR / f"{new_key}.py"
    path.write_text(TEMPLATE.format(module=module, cls=cls, key=new_key))
    return path

def list_proxy_keys() -> list[str]:
    if not CUSTOM_CHANNELS_DIR.exists():
        return []
    return [p.stem for p in CUSTOM_CHANNELS_DIR.glob("*.py") if p.stem != "__init__"]

def delete_proxy(key: str) -> bool:
    path = CUSTOM_CHANNELS_DIR / f"{key}.py"
    if path.exists():
        path.unlink()
        return True
    return False
```

### 2. `src/qwenpaw/app/routers/config.py`

**2a. POST `/channels/{channel_name}/duplicate`** — новый endpoint:

```python
@router.post("/channels/{channel_name}/duplicate")
async def duplicate_channel(request: Request, channel_name: str):
    if channel_name not in BUILTIN_CHANNEL_KEYS or channel_name == "console":
        raise HTTPException(400, "Can only duplicate built-in channels (except console)")
    if channel_name not in CHANNEL_CLASS_MAP:
        raise HTTPException(400, f"Channel '{channel_name}' does not support duplication")

    existing = list_proxy_keys()
    idx = 1
    while f"{channel_name}_copy_{idx}" in existing:
        idx += 1
    new_key = f"{channel_name}_copy_{idx}"

    generate_proxy(channel_name, new_key)

    # Сохранить пустой конфиг для текущего агента (enabled: false)
    agent = await get_agent_for_request(request)
    src_cfg = _get_channel_config_dict(agent.config.channels, channel_name)
    new_cfg = {**src_cfg, "enabled": False, "duplicated_from": channel_name}
    setattr(agent.config.channels, new_key, new_cfg)
    save_agent_config(agent.agent_id, agent.config)
    schedule_agent_reload(request, agent.agent_id)
    return {"key": new_key}
```

**2b. DELETE `/channels/{channel_name}`** — новый endpoint:

```python
@router.delete("/channels/{channel_name}")
async def delete_custom_channel(request: Request, channel_name: str):
    if channel_name in BUILTIN_CHANNEL_KEYS:
        raise HTTPException(400, "Cannot delete built-in channels")
    delete_proxy(channel_name)
    agent = await get_agent_for_request(request)
    extra = dict(agent.config.channels.__pydantic_extra__ or {})
    extra.pop(channel_name, None)
    save_agent_config(agent.agent_id, agent.config)
    schedule_agent_reload(request, agent.agent_id)
    return {"deleted": channel_name}
```

Новая функция:

```python
def make_agent_aware_process(runner, workspace):
    local_process = make_process_from_runner(runner)
    async def process(request):
        target = getattr(request, "channel_meta", None)
        target_agent = getattr(target, "_target_agent", None) if target else None
        # Пустой target_agent = текущий агент (default поведение)
        if target_agent and target_agent != workspace.agent_id:
            try:
                target_ws = await workspace._manager.get_agent(target_agent)
                target_runner = target_ws.runner
                if target_runner:
                    return await make_process_from_runner(target_runner)(request)
            except Exception:
                logger.warning("Failed to route to agent %s, using local", target_agent)
        return await local_process(request)
    return process
```

## Риски

1. **`__pydantic_extra__` мутация при delete** — pydantic v2, нужен rebuild или пересоздание `ChannelConfig`
2. **Registry задержка** — reload агента занимает 1-3 секунды для подхвата нового proxy
3. **Два агента включают один proxy** — конфликт polling/webhook. UI не должен это запрещать, но нужно warning
4. **Orphan proxy-файлы** — если удалить канал из UI, `.py` тоже удаляется. Но ручное удаление из конфига оставит файл
