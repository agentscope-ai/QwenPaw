# 设计理念与架构哲学

## 我们在构建什么

今天的大模型已经很强了，能理解复杂指令，能多步推理，能生成高质量内容。但模型本身只会一件事：给一段输入，生成一段输出。它没有记忆，不知道自己是谁，没法操作任何东西，也不会在两次对话之间记住你说过什么。聊完就忘，下次又从头开始。

QwenPaw 要做的事情很明确：让大模型从一个「内容生成引擎」变成一个能**持续存在、持续成长**的智能体。

我们的思路是构建一个 **Agent OS**，一个专门给 AI 智能体用的操作系统。操作系统给进程提供独立的内存空间、文件系统、输入输出、进程间通信、权限管理；智能体想要真正跑起来，也需要这一整套对应的基础设施。QwenPaw 就是这一整套。

业界管模型之外的这层基础设施叫Agent Harness。模型管推理和生成，Harness 管把模型的输出变成实际的行为。模型是可以换的大脑，QwenPaw 是让这个大脑活在真实世界里的一切。

---

## 智能体的五个存在条件

一个 AI 智能体想在用户的世界里真正「存在」，至少得满足什么条件？

| 条件                | 含义                                   |
| ------------------- | -------------------------------------- |
| 工作区（Workspace） | 有自己的地盘，状态能持久保存           |
| 人设（Persona）     | 知道自己是谁，行为稳定，风格一致       |
| 记忆（Memory）      | 能积累经验，能跨越时间理解上下文       |
| 能力（Capability）  | 能感知环境，也能对环境做出改变         |
| 连接（Connection）  | 能跟人、跟其他智能体、跟外部系统打交道 |

QwenPaw 的每个核心子系统，都在解决上面某个条件。

## 工作区：智能体的地盘

QwenPaw 里每个智能体有自己的工作区。

工作区（Workspace）是 Agent OS 里**承载智能体的容器**。智能体的所有东西，人设、记忆、技能、聊天记录、定时任务、配置，全都放在工作区里。它本质上就是一个文件目录，里面全是人能直接看懂的文件。

我们管这个做法叫**状态外化**：把智能体的一切都摊开放成文件，不藏进数据库。好处很实在。想知道智能体为什么这么做？打开文件看就行。想改它的行为？编辑 Markdown。想备份、迁移？拷贝目录。而且文件会随着使用不断变化，不是死模板，是一个活的、会生长的空间。

工作区也让智能体变成了可以打包传递的东西。调教好一个智能体，打个包分享出去，别人导入就能用。多个智能体跑在同一个 QwenPaw 里时，各有各的工作区，隔离是默认的，要共享得明确声明。

## 人设：智能体是谁

人设决定了智能体拿到一条消息之后，怎么理解、怎么回应。

在 QwenPaw 里，人设就是工作区里的一组 Markdown 文件。智能体启动的时候把这些文件读进来，拼进系统提示词。不同的文件管不同的事：有的管底层人格和价值观，有的管名字、风格、专长，有的管具体的工作流程和协作方式。全用自然语言写，大模型天然就能读懂，不需要转什么格式。改了文件马上生效，不用重启。

随着跟用户互动得越来越多，智能体还能自己丰富自己的画像。人设不是一张贴死的标签，它会跟着经验一起长。

## 记忆：从一问一忘到越用越懂你

没有记忆的 AI 助手，每次开聊都跟初次见面一样。QwenPaw 把记忆分成两层。

**工作记忆**，就是当前这轮对话的上下文，模型直接能看到的东西。上下文窗口有大小限制，快满的时候 QwenPaw 会做智能压缩，尽量多留关键信息。工作记忆是临时的，对话结束就没了。

**长期记忆**，存进工作区文件里的东西都是长期记忆：对话中提炼出来的用户偏好和关键事实、学到的技能知识、干活时写出来的脚本和文档。光记住还不够，还得会整理和想起来。记忆多了要归纳整理，新对话开始时要能根据话题自动把相关记忆找出来、塞回上下文。

两层记忆之间不停在**来回流动**：对话里的重要信息沉淀成长期记忆，长期记忆里相关的东西被拉回来帮助当前推理。这个循环一直在转，智能体就越来越了解你。

再往远了说，记忆的终极形态不是被动的存储，而是一个**主动的认知引擎**。智能体自己从记忆里发现规律、预判需求，不等你开口就主动来找你。

## 能力：模型负责想，工具负责干

大模型是大脑，管思考和决策。但跟环境打交道的活，读文件、跑命令、上网查东西，得靠工具。QwenPaw 提供了四种能力接入方式：

- 内置工具（Tools）：读写文件、执行命令、浏览网页、截屏看桌面等等，基本操作，开箱就能用
- 技能（Skills）：用自然语言写的能力描述，外加可选的脚本。智能体读完就算学会了。运行时随装随卸、按频道灵活配置，谁都能写、能分享
- MCP：通过 [MCP 协议](https://modelcontextprotocol.io) 接外部工具服务，支持热插拔，断了自动重连
- 插件（Plugins）：开发者往里注入自己写的工具，不用改核心代码

这四种能力不分高低，大模型通过循环**统一调度**：看、想、做、回头看一眼，如此循环直到事情办完。不管某个能力是从哪来的，模型调用起来都一样，Agent OS 把来源差异屏蔽掉了。

这些能力之间的边界也不是死的。智能体在干活过程中，可以把跑通的流程固化成新的技能。它能在实践中**给自己造新能力**，整个能力体系是会自己长的。

---

## 连接：跟人、跟智能体、跟世界打交道

智能体不该被关在一个界面里。QwenPaw 提供了多个维度的连接：

- 频道（Channels）：同时接入钉钉、飞书、微信、Discord、Telegram 等十余种平台，用户在哪儿聊，智能体就在哪儿答
- ACP：通过 [Agent Communication Protocol](https://agentcommunicationprotocol.dev) 跟外部智能体互联互通。别人可以发现和调用你的智能体，你的智能体也能主动找外面的智能体协作
- 控制台和 API：Console 提供可视化管理，RESTful API 方便把智能体集成进更大的系统
- 人机协同：审批确认、操作把关、反馈纠偏。人不只是用户，也是智能体运行时的协作者。拿不准的事情，智能体会来问你

这一层的核心思路就是**开放**。能接任何平台，能跟任何走标准协议的系统协作，同时人始终在回路里。

## 它们是怎么一起工作的

上面讲的不是一堆独立的功能，而是**一个整体**。一次典型的交互是这样串起来的：

智能体在自己的工作区里运行，人设文件告诉它用什么身份和风格来回复。消息进入模型之前，记忆系统先从长期记忆里把跟当前话题有关的东西找出来，塞进上下文。模型拿到完整上下文后开始推理，决定需要调哪些工具或技能，也许是读个文件，也许是调个 MCP 服务，也许是跑一段技能里的流程。每次工具调用都过一遍安全层。任务做完后，对话里有价值的信息被自动提炼，沉淀回长期记忆。智能体对你的了解，又多了一层。
