# CoPaw vs OpenClaw : Comparaison des fonctionnalités

| Catégorie                | Dimension                            | OpenClaw                                                                                                                                                                   | CoPaw                                                                                                                                                                                                                                         |
| :----------------------- | :----------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Stack technique          | Langage principal                    | TypeScript / Node.js                                                                                                                                                       | Python                                                                                                                                                                                                                                        |
|                          | Framework agent                      | Intégré en profondeur avec le SDK pi-agent comme moteur d'agent principal                                                                                                  | • Framework multi-agents AgentScope<br>• AgentScope-Runtime                                                                                                                                                                                   |
|                          | Système de mémoire                   | Mémoire basée sur des fichiers (JSONL + MD)                                                                                                                                | Système de mémoire basé sur des fichiers alimenté par AgentScope-AI/ReMe                                                                                                                                                                      |
| Expérience utilisateur   | Installation                         | • npm install -g openclaw<br>• Assistant CLI openclaw onboard                                                                                                              | • Installation par script en une ligne<br>• Alternative : pip install copaw<br>• Installation Docker                                                                                                                                          |
|                          | Plateformes supportées               | macOS / Linux / Windows (WSL2)                                                                                                                                             | macOS / Linux / Windows (PowerShell/CMD)                                                                                                                                                                                                      |
|                          | Auto-hébergement / Déploiement cloud | Auto-hébergé via Docker                                                                                                                                                    | • Auto-hébergé via Docker (agentscope/copaw:latest, avec Console Web)<br>• Déploiement cloud en un clic                                                                                                                                       |
|                          | Support des modèles locaux           | Configurer les points de terminaison Ollama / llama.cpp via le fichier de config                                                                                           | Intégration à la demande via --extras lors de l'installation :<br>• --extras llamacpp (multiplateforme)<br>• --extras mlx (Apple Silicon)<br>• --extras ollama (multiplateforme, nécessite le service Ollama)                                 |
|                          | Support des Skills                   | • Supporte les Skills locales<br>• Installation depuis ClawHub                                                                                                             | • Supporte les Skills locales<br>• Import direct depuis plusieurs Skills Hubs communautaires<br>(Actuellement supportés : https://skills.sh/..., https://clawhub.ai/..., https://skillsmp.com/..., https://github.com/...)                    |
|                          | Intégrations de canaux               | WhatsApp, Telegram, Slack, Discord, iMessage, Signal, Teams, Matrix, Zalo, etc. ; support de canaux extensible                                                             | DingTalk, Feishu, QQ, Discord, iMessage, etc. ; support de canaux extensible                                                                                                                                                                  |
| Écosystème communautaire | Licence open source                  | MIT (entièrement ouvert)                                                                                                                                                   | Apache 2.0 (adapté aux entreprises)                                                                                                                                                                                                           |
|                          | Extensibilité                        | Intégrations de Skills communautaires supportées                                                                                                                           | Intégrations de Skills communautaires supportées                                                                                                                                                                                              |
| Feuille de route         | Modèle de sécurité                   | • Renforcement de la sécurité piloté par la communauté (propositions de sandbox WASM, protection des clés API, etc.)<br>• Marketplace ClawHub avec scan VirusTotal complet | • Piloté par la communauté<br>• Feuille de route officielle : tool-sandbox                                                                                                                                                                    |
|                          | Collaboration multi-agents           | Piloté par la communauté                                                                                                                                                   | Hérite du support multi-agents natif via le framework AgentScope                                                                                                                                                                              |
|                          | Architecture cloud-native            | Piloté par la communauté                                                                                                                                                   | • Piloté par la communauté<br>• Feuille de route officielle : étendre continuellement les capacités cloud via AgentScope Runtime, couvrant le calcul cloud, le stockage et un large écosystème de services cloud                              |
|                          | Collaboration modèle grand-petit     | Non mentionné explicitement dans les docs officiels                                                                                                                        | Feuille de route officielle : LLMs locaux légers pour gérer les données sensibles à la confidentialité ; tâches complexes (ex. planification, code) routées vers des LLMs cloud puissants — équilibre entre sécurité, performance et capacité |
|                          | Entraînement de modèles locaux       | Piloté par la communauté                                                                                                                                                   | Feuille de route officielle : entraîner et affiner des LLMs déployables localement spécifiquement pour les Skills natives principales de CoPaw et les tâches courantes                                                                        |
|                          | Interaction multimodale              | Piloté par la communauté                                                                                                                                                   | Feuille de route officielle : support de l'interaction vocale et vidéo                                                                                                                                                                        |
|                          | Écosystème communautaire             | Piloté par la communauté ; ClawHub en expansion continue                                                                                                                   | Feuille de route officielle : enrichir continuellement le dépôt AgentScope-Skills et améliorer la découvrabilité des Skills de qualité                                                                                                        |
